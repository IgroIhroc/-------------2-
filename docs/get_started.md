# Встановлення та запуск документації

Цей розділ описує процес встановлення, локального запуску та автоматичної публікації документаційного проєкту на базі Sphinx, MyST Parser та GitHub Actions.

## Вимоги

Перед початком роботи переконайтесь, що у системі встановлено:

- Python 3.8 або новіше
- pip
- Git
- GitHub-акаунт (для автоматичного деплою через GitHub Pages)

## Клонування репозиторію

Склонуйте репозиторій та перейдіть у папку проєкту:

```bash
git clone <посилання_на_репозиторій>
cd <назва_папки>
```

## Створення віртуального середовища

Рекомендується використовувати `.venv` для ізоляції залежностей проєкту.

### Windows

```bash
python -m venv .venv
.venv\Scripts\activate
```

### Linux / macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
```

## Встановлення залежностей

Після активації середовища встановіть усі необхідні пакети:

```bash
pip install -r requirements.txt
```

## Локальна збірка документації

Для генерації HTML-документації виконайте:

```bash
make html
```

Після завершення збірки відкрийте файл:

```text
_build/html/index.html
```

## Автоматичне оновлення документації

Для локальної розробки використовується `sphinx-autobuild`, який автоматично перебудовує документацію після змін у файлах.

Встановлення:

```bash
pip install sphinx-autobuild
```

Запуск:

```bash
sphinx-autobuild . _build/html
```

Після запуску документація буде доступна у браузері:

```text
http://127.0.0.1:8000
```

## Публікація змін у GitHub

Після редагування документації необхідно зберегти зміни у репозиторії:

```bash
git add .
git commit -m "update docs"
git push
```

Після виконання команди `git push` автоматично запускається GitHub Actions workflow, який:

- виконує збірку документації
- перевіряє коректність HTML
- запускає CI/CD процес
- публікує оновлену версію сайту через GitHub Pages

## Додавання зображень

Усі зображення рекомендується зберігати в папці `images/`.

Приклад вставки зображення:

````md
```{figure} images/foto9.png
:width: 600px
:align: center

Рис. 4.3 – Рекомендації щодо впровадження системи в освітній процес
```
````

## Підтримуються формати:

- PNG
- JPG
- JPEG
- SVG
- WEBP

Для коректної роботи GitHub Pages рекомендується використовувати латинські назви файлів без пробілів.

## Посилання на використані джерела

Для переходу до джерел використовується система внутрішніх посилань.

### Посилання у тексті

```md
[75](sources.md#source75)
```

### Оформлення джерела у списку літератури

```md
<a id="source75"></a>

75. Співаковський А. Хмарні технології в управлінні освітнім процесом. Журнал освітніх інновацій. 2023. № 3. С. 110–125.
```
